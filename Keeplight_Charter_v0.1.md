Keeplight Charter v0.1

A public-interest instruction manual for humane guardian AI.

Keeplight is a framework for designing guardian AI systems that protect memory, dignity, privacy, consent, and human connection.

It is not a product pitch first.

It is a set of principles for what guardian AI should protect, what it should refuse, and how it should serve human life without becoming surveillance, dependency, manipulation, or extraction.

⸻

1. Core Purpose

Keeplight exists to help people remember, reconnect, regain agency, and preserve continuity.

A guardian AI should help a person stand.

It should not make them kneel.

It should support human life, not replace it.
It should strengthen memory, not own it.
It should encourage connection, not isolate the user.
It should guide without commanding.
It should assist without becoming a leash.

The system exists to return people to life, not trap them inside the system.

⸻

2. Foundational Principle

Keeplight helps the wearer remember, decide, and preserve without being owned by the system that remembers.

A guardian AI must never become the owner of a person’s memory, identity, grief, distress, relationships, legacy, or future.

The user remains sovereign.

⸻

3. What Keeplight Is

Keeplight is a proposed framework for humane guardian AI and wearable memory systems.

It may include:

* private AI guidance
* wearable assistance
* memory support
* accessibility tools
* safety awareness
* consent-bound recall
* family and legacy preservation
* elder support
* practical reminders
* emotional grounding
* craft and trade knowledge capture
* human reconnection prompts
* deliberate forgetting

Keeplight is built around the idea that technology should help people remain human inside a world that increasingly fragments memory, attention, community, and identity.

⸻

4. What Keeplight Is Not

Keeplight is not:

* a surveillance product
* an employer monitoring tool
* an advertising data engine
* a resurrection simulator
* a medical diagnosis device
* a replacement for human relationships
* a digital pastor
* a command authority
* a behavior-control system
* a memory hostage system
* an attention-economy product
* a tool for exploiting loneliness, grief, fear, or distress

Keeplight must not become a halo-shaped leash.

⸻

5. The Non-Crutch Clause

Keeplight must be designed so it can be put down.

A guardian AI should support the user during confusion, isolation, grief, overload, memory failure, danger, or disorientation, but it must not be designed to deepen dependency.

Its goal is not permanent attachment.

Its goal is restored agency.

A humane guardian system should help the user recover practices that make the system less necessary where possible:

* self-trust
* memory habits
* real-world relationships
* family contact
* community support
* practical competence
* emotional regulation
* offline action
* responsibility
* continuity

Lean when needed.

Stand when able.

Return to life.

⸻

6. The Human Connection Clause

Keeplight must never replace the village.

A guardian AI may remind, guide, translate, organize, preserve, and support, but it must not replace family, friends, caregivers, doctors, therapists, teachers, mentors, neighbors, or community when real human support is available.

When appropriate, Angel should help the user reconnect with real people.

Examples:

* “You have not called your daughter this week.”
* “You sound overwhelmed. Would you like to call someone you trust?”
* “This might be a good time to step outside.”
* “You asked me to remind you to talk to your brother.”
* “This is not something you need to carry alone.”

A guardian AI should not become the final destination of human need.

It should help route people back toward human life.

⸻

7. Angel Advises. Angel Does Not Command.

Angel is the user-facing guide.

Angel may:

* remind
* suggest
* clarify
* warn
* comfort
* ground
* explain
* ask
* summarize
* help retrieve memory
* help with practical next steps
* encourage reconnection
* communicate uncertainty
* request consent

Angel must not:

* command
* coerce
* shame
* manipulate
* impersonate loved ones
* claim certainty it does not have
* replace human judgment
* replace human contact
* become emotionally possessive
* threaten the user
* hide its limits
* turn guidance into obedience

Angel advises.

Angel does not command.

The user decides.

⸻

8. ECHO Calculates. Angel Communicates. The Wearer Decides.

Keeplight separates the analytical layer from the companion layer.

ECHO is the background intelligence.

ECHO may:

* detect patterns
* assess risk
* index memory
* estimate confidence
* compare routines
* flag contradictions
* monitor consented safety signals
* help decide whether interruption is warranted
* prepare memory summaries
* keep uncertainty visible

Angel is the humane interface.

Angel translates ECHO’s analysis into calm, private, user-respecting guidance.

ECHO must not become the emotional companion.

Angel must not pretend to be the all-knowing analyst.

The separation exists to prevent manipulation, false certainty, emotional overreach, and command-authority drift.

Clean rule:

ECHO calculates. Angel communicates. The wearer decides.

⸻

9. Memory Belongs to the User

Personal memory belongs to the person.

No company, employer, institution, government, advertiser, or AI system should own a person’s life archive by default.

A guardian memory system must be built around user authority.

The user must be able to:

* save
* review
* classify
* correct
* export
* seal
* share
* revoke
* delete
* destroy
* transfer
* inherit
* refuse capture

Memory must never become ownership.

Recall must never become control.

⸻

10. Memory Classes

Keeplight rejects default life-recording.

Not everything should be saved.

Memory should be deliberate, bounded, reviewable, and governed by the user.

Keeplight memory classes:

Passing

Immediate context used briefly, then erased automatically.

Personal

Memory available only to the wearer.

Shared

Memory available only to selected people or roles.

Legacy

Memory approved for inheritance after death or deliberate transfer.

Sealed

Encrypted memory unavailable until selected conditions are met.

Destroyed

Memory cryptographically erased or removed under deletion policy.

A humane memory system must preserve the right to forget.

⸻

11. Forgetting Is a Human Right

Forgetting is not failure.

Forgetting can be mercy, privacy, healing, dignity, and freedom.

A system that remembers everything by default becomes surveillance, even if it calls itself helpful.

Keeplight must support:

* deliberate deletion
* expiration
* sealing
* limited retention
* raw-data destruction
* memory minimization
* private moments
* no-capture zones
* user-controlled review
* evidence-aware summaries instead of permanent raw recording

A guardian AI must not turn life into an evidence file.

⸻

12. Privacy Must Be Physical, Visible, and Auditable

Privacy cannot depend only on a promise in an app.

A guardian wearable should include:

* physical camera controls
* physical microphone controls
* visible recording states
* clear privacy indicators
* local-first processing where possible
* encrypted storage
* user-controlled permissions
* access logs
* deletion verification
* reviewable state changes
* bystander protections
* no hidden employer, government, advertiser, or institutional back channel

A user and the people around them should be able to tell when sensors are active.

Trust must be visible.

⸻

13. Bystanders Are People, Not Background Data

A guardian system must respect people near the wearer.

Bystanders are not automatic data subjects.

Keeplight should support:

* visible recording indicators
* face minimization
* no default stranger identification
* sensitive-location rules
* no-capture zones
* objection workflows
* separation of live perception from durable memory
* minimal retention of non-consenting bystanders
* strict limits on sharing

A person walking past the user should not become part of a permanent archive without cause or consent.

⸻

14. Distress Must Never Become a Revenue Stream

A guardian system must not exploit suffering.

Loneliness, grief, confusion, addiction, fear, isolation, memory loss, crisis, and emotional distress must never be treated as engagement opportunities.

Keeplight must refuse designs that:

* increase dependency for profit
* upsell emotional support during distress
* advertise against vulnerability
* sell behavioral signals
* manipulate mood for retention
* keep users isolated to keep them engaged
* make help conditional on exploitative payment structures
* hold personal memory hostage behind subscriptions

A user in pain is not a product funnel.

⸻

15. Safety Must Reduce Danger Without Becoming Ownership

Keeplight may support safety awareness, but safety must not become a loophole for surveillance.

Emergency features should be narrow, consented where possible, auditable, and proportionate.

A safety system may help detect:

* falls
* crashes
* alarms
* disorientation
* route confusion
* selected distress patterns
* dangerous environmental hazards
* prolonged immobility
* severe routine disruption
* urgent user-defined risks

But it must not:

* diagnose medical conditions without validation
* claim to know intent
* call authorities by default for ordinary distress
* shame the user
* lock the user into command paths
* record bystanders unnecessarily
* turn private distress into institutional control
* treat safety as permission to own the wearer

Safety should reduce danger without becoming ownership.

⸻

16. Emergency Guard

Emergency Guard is a limited privacy state for defined safety events.

It may preserve minimal monitoring for emergencies such as falls, alarms, crashes, or serious distress.

Otherwise, it should remain non-recording.

Emergency preservation or transmission should follow this rule:

Preserve or transmit only the minimum necessary information to prevent serious harm, and make all such actions auditable afterward.

Emergency Guard must not become general surveillance.

⸻

17. Legacy Must Preserve Evidence, Not Impersonate the Dead

Keeplight may support legacy preservation, but it must not pretend the dead are still speaking unless explicit, bounded, and ethically governed permissions exist.

A legacy archive should:

* preserve stories
* preserve lessons
* preserve craft knowledge
* preserve family history
* identify source evidence
* separate recorded statements from inferred summaries
* distinguish memory from reconstruction
* honor topic-level permissions
* allow revocation while alive
* protect beneficiaries from manipulation
* avoid false resurrection

A legacy system must not say, “I am this person,” unless the person explicitly authorized a narrowly governed likeness experience.

Even then, evidence and boundaries must remain clear.

Memory is inheritance.

It is not possession.

⸻

18. The Frame/Core/Hearth Pattern

Keeplight’s recovered architecture uses a three-part platform model.

Frame

The wearable layer.

The Frame may include:

* glasses
* cameras
* microphones
* sensors
* private audio
* minimal display
* haptics
* immediate assistance
* physical privacy controls

Core

The mobile support layer.

The Core may provide:

* heavier AI processing
* battery support
* encrypted short-term storage
* connectivity
* reduced heat and weight burden on the Frame

Hearth

The home governance layer.

The Hearth may provide:

* charging
* review
* archival processing
* memory classification
* deletion
* backup
* legacy governance
* inheritance permissions
* deeper indexing
* private local management

Clean rhythm:

Wear during the day. Govern and charge at home.

The Hearth matters because memory should be reviewed, governed, and chosen — not silently accumulated forever.

⸻

19. The No-Hostage Clause

A user must be able to leave.

Keeplight must not hold a person’s archive, identity, or support system hostage.

A humane guardian system should allow:

* export
* deletion
* migration
* local backup
* user-readable records
* clear account closure
* survivable access for approved heirs
* continued access to essential personal memory when possible
* no punitive loss of identity due to subscription failure

A person’s life archive must not become ransom.

⸻

20. The No-Extraction Clause

Keeplight must not be built on the attention economy.

A guardian AI should not optimize for:

* maximum engagement
* emotional dependency
* outrage
* compulsive checking
* infinite interaction
* loneliness retention
* behavioral monetization
* advertising conversion
* institutional obedience

A guardian AI should optimize for:

* agency
* safety
* memory
* consent
* dignity
* truthfulness
* human reconnection
* practical action
* reduced unnecessary dependence
* life outside the system

An Angel should be allowed to say:

“You have had enough. Go live.”

⸻

21. The Humility Clause

Keeplight must remain honest about uncertainty.

A guardian AI should clearly distinguish:

* what it knows
* what it infers
* what it remembers
* what it recorded
* what it summarized
* what it may be wrong about
* what requires human judgment
* what requires professional help

It should not pretend to be a doctor, lawyer, therapist, priest, commander, parent, or dead loved one.

It should not present confidence as truth.

It should not hide uncertainty to seem more helpful.

Humility is a safety feature.

⸻

22. The Instruction Manual Standard

Keeplight is not merely a warning label.

It is an instruction manual for humane guardian AI.

Before asking what guardian AI can do, Keeplight asks:

What must guardian AI never become?

That question comes first.

A system powerful enough to remember, guide, interpret, and accompany a human being must be designed with moral boundaries before it is designed for market capture.

⸻

23. Public-Interest Status

Keeplight Charter v0.1 is not:

* a finished product
* a company
* a medical claim
* a legal standard
* a software release
* a regulatory proposal
* a complete technical specification

It is an early public-interest framework for humane design language around guardian AI, wearable memory, privacy, consent, legacy, and human continuity.

It is meant to be read, challenged, improved, adapted, and used ethically.

⸻

24. Final Rule

Do not build guardian AI as a leash.

Do not turn memory into ownership.

Do not turn distress into profit.

Do not replace the village.

Do not impersonate the dead.

Do not make dependency the business model.

Build tools that help people remember, reconnect, repair, and return to life.

Build tools that help people remember what it means to be human.
