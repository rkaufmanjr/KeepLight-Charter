Keeplight Charter

A public-interest charter for guardian AI that protects memory, dignity, privacy, and human connection.

Keeplight is an open concept framework for humane guardian AI: systems designed to help people remember, reconnect, regain agency, and preserve continuity without becoming surveillance, dependency, manipulation, or extraction.

This repository begins with the Keeplight Charter: an instruction manual for what guardian AI should protect, what it should refuse, and how it should serve human life.

⸻

Core Premise

Guardian AI should help people stand, not make them kneel.

Keeplight is not a proposal for an addictive companion, corporate surveillance wearable, resurrection simulator, employer monitoring tool, advertising engine, or obedience system.

Keeplight is a human-controlled continuity framework built around:

* memory
* consent
* privacy
* dignity
* practical guidance
* human reconnection
* deliberate forgetting
* legacy protection
* non-command AI support

The system exists to return people to life, not trap them inside the system.

⸻

The Keeplight Principle

Keeplight helps the wearer remember, decide, and preserve without being owned by the system that remembers.

A guardian AI should never become the owner of a person’s memory, identity, distress, relationships, or future.

⸻

Essential Rules

1. The User Owns the Memory

Personal memory belongs to the person.

No company, employer, institution, government, advertiser, or AI system should own a person’s life archive by default.

⸻

2. Angel Advises. Angel Does Not Command.

The user-facing guide may suggest, remind, clarify, warn, and support.

It must not command, manipulate, shame, coerce, impersonate loved ones, replace human contact, or pretend to know more than it does.

⸻

3. ECHO Calculates. Angel Communicates. The Wearer Decides.

The analytical layer and the companion layer must remain separate.

ECHO may detect patterns, estimate risk, index memory, and provide confidence-ranked observations.

Angel may communicate those observations calmly and humanely.

The wearer remains sovereign.

⸻

4. Distress Must Never Become a Revenue Stream

A guardian system must not exploit loneliness, grief, confusion, addiction, fear, isolation, memory loss, or emotional distress for profit.

A user in pain is not an engagement opportunity.

⸻

5. Forgetting Must Be Allowed

Not everything should be saved.

A humane memory system must support deliberate forgetting, deletion, sealing, expiration, and limited retention.

Memory without forgetting becomes surveillance.

⸻

6. Privacy Must Be Physical, Visible, and Auditable

Privacy cannot depend only on a promise in an app.

A guardian wearable should include visible privacy states, hardware-level sensor controls, clear user authority, and reviewable access logs.

⸻

7. The System Must Encourage Human Connection

A guardian AI should not replace family, friends, doctors, therapists, teachers, mentors, caregivers, neighbors, or community.

When appropriate, it should help the user reconnect with real people.

⸻

8. The System Must Be Designed So It Can Be Put Down

Keeplight is not meant to become a permanent crutch.

Its purpose is to help people regain grounding, memory, agency, and connection.

A humane system should reduce dependency where possible.

⸻

Recovered Architecture

The recovered Keeplight architecture is built around a three-part platform:

Frame

The wearable glasses layer: perception, private interface, minimal HUD, audio, haptics, immediate assistance, and physical privacy controls.

Core

A companion compute and battery module for heavier processing, encrypted short-term storage, connectivity, and reduced heat/weight burden on the glasses.

Hearth

A private home dock for charging, review, archival processing, backup, deletion, memory classification, and legacy governance.

The clean system rhythm:

Wear during the day. Govern and charge at home.

⸻

Memory Classes

Keeplight uses deliberate memory classes rather than default life-recording:

* Passing — immediate context, erased automatically
* Personal — available only to the wearer
* Shared — available only to selected people or roles
* Legacy — approved for inheritance after death or deliberate transfer
* Sealed — encrypted and unavailable until selected conditions are met
* Destroyed — cryptographically erased or removed under deletion policy

The system should not store a raw life-record by default.

⸻

What Keeplight Is Not

Keeplight is not:

* a surveillance product
* an employer monitoring system
* an advertising data engine
* a resurrection simulator
* a medical diagnosis device
* a magic truth machine
* a replacement for human relationships
* a system for institutional control
* a tool for exploiting loneliness
* a memory hostage system

⸻

What Keeplight Is For

Keeplight is for exploring how guardian AI could support:

* memory aid
* elder care
* Alzheimer’s and dementia support research
* accessibility
* veteran support
* emotional grounding
* daily planning
* social reconnection
* practical reminders
* family legacy
* craft and trade teaching
* knowledge preservation
* privacy-respecting personal continuity

This is not a claim that Keeplight can cure, diagnose, or replace professional care.

It is a framework for asking how such systems should be designed if they are built.

⸻

Foundational Design Question

Before asking what guardian AI can do, Keeplight asks:

What must guardian AI never become?

That question comes first.

⸻

Status

This repository is an early public-interest concept and design charter.

It is not a finished product, company, medical device, legal standard, or software release.

It is a starting point for humane design language around guardian AI, wearable memory, consent, privacy, and human continuity.

⸻

Invitation

Use this framework. Challenge it. Improve it. Fork it. Critique it. Adapt it ethically.

But do not build guardian AI as a leash.

Do not turn memory into ownership.

Do not turn distress into profit.

Do not replace the village.

Build tools that help people remember what it means to be human.
